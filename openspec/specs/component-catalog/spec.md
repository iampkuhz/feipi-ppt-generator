# Component Catalog

## Purpose

Keep components discoverable, validated, and testable.

## Requirements

- Every component has a registry entry and props schema.
- New components include examples and tests.
- Registry validation is required before rendering.

## Non-goals

- Allowing unregistered components in production specs.

## Examples

`PageTitle`, `BadgePill`, `IconLabel`, `MetricBlock`, `IconGrid`, and `ChartCard`.

## Validation

Run registry tests and component schema validation.

# Pattern Library

## Purpose

Define whole-slide layout patterns and narrative slots.

## Requirements

- Patterns are Layer4 constructs.
- Patterns compose registered components.
- Pattern additions require examples and QA expectations.

## Non-goals

- Hard-coding a specific deck or template page.

## Examples

`CoverSlide`, `TitleContentSlide`, and `ThankYouSlide`.

## Validation

Validate examples and renderer smoke output.
